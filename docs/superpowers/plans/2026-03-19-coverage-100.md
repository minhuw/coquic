# 100 Percent Coverage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Raise the current LLVM source-based coverage totals to 100% for line, branch, and region coverage without changing intended QUIC behavior.

**Architecture:** Close the remaining gaps with test-first additions in the current GoogleTest suites, then apply only small behavior-preserving cleanups where coverage is blocked by provably unreachable fallback code. Most of the remaining work is concentrated in `src/quic/connection.cpp`, with smaller targeted gaps in `src/quic/demo_channel.cpp`, `src/quic/core.cpp`, and `src/quic/tls_adapter_quictls.cpp`.

**Tech Stack:** Zig build system, C++20, GoogleTest, LLVM `llvm-cov` / `llvm-profdata`, quictls
